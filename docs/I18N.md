# Internationalization (i18n)

The UI text is centralized in the `translations` object in `app.js`. The app renders static copy
via `data-i18n` attributes in `index.html` and dynamic copy through the `t()` helper.

## Adding a new language
1. Add a new language key to `translations` (e.g. `fr`).
2. Provide translations for every key in the English catalog.
3. Update the language selector labels in `renderLanguageOptions()` if needed.
4. Verify the UI end-to-end with the new language selected.

## Adding new UI text
- **Static HTML:** add a new key to `translations` and bind it with `data-i18n`.
- **Dynamic JS:** reference `t("path.to.key")` and avoid hard-coded strings.

## Storage
The app stores the selected language in `localStorage` under `ryf:lang`.
